## 🔧 Critical Fixes Implemented

### 1. Enhanced STS Parser (`preprimer/parsers/sts_parser.py`)

**Problem**: Could not parse real-world me-PCR/merPCR files
- ❌ Required exact 3-column format with header
- ❌ Failed on header-less files (common in me-PCR output)
- ❌ Failed on 4-column extended format (with product size)

**Solution**: Completely rewritten parser with intelligent format detection
- ✅ Auto-detects 3 or 4-column format
- ✅ Auto-detects header presence vs. header-less
- ✅ Uses DNA sequence pattern matching to distinguish headers from data
- ✅ Extracts and uses product size when available
- ✅ Maintains backward compatibility

**Impact**:
- Fixed 6 previously failing/skipped tests
- Enabled full merPCR integration
- Made PrePrimer compatible with real me-PCR output files

### 2. Enhanced STS Writer (`preprimer/writers/sts_writer.py`)

**Problem**: Always output 3-column format, incompatible with merPCR

**Solution**: Intelligent auto-detection with configuration options
- ✅ Auto-detects when to include SIZE column (based on amplicon length availability)
- ✅ Configurable via `include_size` parameter
- ✅ Configurable header inclusion via `include_header` parameter
- ✅ Outputs correct format for target tool (me-PCR vs. merPCR)

**Impact**:
- Full merPCR pipeline now working
- Better tool compatibility

### 3. Enhanced STS Validator (`tests/validation/validator.py`)

**Problem**: Only validated 3-column format

**Solution**: Flexible validation for both formats
- ✅ Detects and validates both 3 and 4-column formats
- ✅ Auto-detects header presence
- ✅ Reports format type in statistics
- ✅ Validates column count consistency
