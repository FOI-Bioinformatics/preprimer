# 🧬 Supported Formats Guide

Complete guide to all input and output formats supported by PrePrimer.

## 📥 **Input Formats**

PrePrimer supports three major primer design tool formats for input.

### **VarVAMP Format**

**Description:** Complete 13-column tab-separated format from the VarVAMP SADDLE algorithm for variant-aware primer design.

**File Extensions:** `.tsv`

**Format Characteristics:**
- Complete 13-column specification with comprehensive metadata
- Full IUPAC degenerate nucleotide support (R, Y, S, W, K, M, B, D, H, V, N)
- Includes amplicon length, quality metrics, and scoring information
- Compatible with VarVAMP SADDLE algorithm for variant-aware designs
- Validated with official schemes including VarVAMP HAV with degenerate primers

#### **File Structure** (Complete 13-column specification)
```tsv
amplicon_name	amplicon_length	primer_name	pool	start	stop	seq	size	gc_best	temp_best	mean_gc	mean_temp	score
NC_012920.1_1	547	FW_1	1	16400	16420	GATCACAGGTCTATCACCCT	20	55.0	58.5	55.0	58.5	92.0
NC_012920.1_1	547	RW_1	1	377	398	GCTGGAGCCGGAGCACCCTAT	21	61.9	60.2	61.9	60.2	94.0
HAV_0	1426	HAV_0_LEFT_1	1	164	186	TGNCCAACYGARTCHGAAGCATT	23	39.1	60.0	39.1	60.0	90.0
HAV_0	1426	HAV_0_RIGHT_1	1	1567	1590	ACACCAAGVCCATTYGGATGRATCC	25	44.0	60.0	44.0	60.0	88.0
```

#### **Required Columns** (Full specification)
- `amplicon_name` - Unique amplicon identifier
- `amplicon_length` - Amplicon length in base pairs
- `primer_name` - Unique primer identifier
- `pool` - Pool assignment (integer)
- `start` - Primer start coordinate (1-based)
- `stop` - Primer end coordinate (1-based, inclusive)
- `seq` - Primer sequence with IUPAC degenerate codes (5' to 3')
- `size` - Primer length in base pairs
- `gc_best` - Best GC content percentage
- `temp_best` - Best melting temperature (°C)
- `mean_gc` - Mean GC content percentage
- `mean_temp` - Mean melting temperature (°C)
- `score` - Primer quality score

#### **Example Usage**
```bash
# Auto-detect VarVAMP format
preprimer convert --input varvamp_output.tsv --output-dir schemes/ --output-formats artic

# Force VarVAMP format
preprimer convert --input ambiguous_file.txt --format varvamp --output-formats artic
```

### **ARTIC Format (primalscheme3 Output)**

**Description:** BED format generated by [primalscheme3](https://github.com/artic-network/primalscheme3) for tiled amplicon sequencing workflows, following the [official primerscheme specification](https://github.com/artic-network/primerscheme-specs).

**File Extensions:** `.bed`, `primer.bed`

**Format Characteristics:**
- Extended BED format with 7 columns (sequence included) following [primal-page specifications](https://github.com/artic-network/primal-page)
- Compatible with articbedversion v2.0 and v3.0 standards
- 0-based coordinates (BED convention) with proper coordinate system handling
- Part of official primerscheme directory structure: `primer.bed` + `reference.fasta` + `info.json`
- Generated by primalscheme3 with full [ARTIC Network](https://github.com/artic-network) compatibility
- Validated with official schemes from [PrimerSchemes Labs](https://labs.primalscheme.com/) repository
- Compatible with [amplicon-nf](https://github.com/nf-core/amplicon-nf) analysis pipelines
- Supports both single and multiple reference architectures

#### **File Structure** (7-column extended BED format)
```bed
# artic-bed-version v3.0
MN908947.3	47	78	SARS-CoV-2_1_LEFT	1	+	CTCTTGTAGATCTGTTCTCTAAACGAACTTT
MN908947.3	419	447	SARS-CoV-2_1_RIGHT	1	-	AAAACGCCTTTTTCAACTTCTACTAAGC
NC_009942.1	8	30	WNVUS1_1_LEFT_1	1	+	GCCTGTGTGAGCTGACAAACTT
reference	14	34	reference_1_LEFT_1	1	+	CGGTTCAGGCTTCACCACAG
```

#### **Column Specification** (Extended BED format)
1. `chrom` - Reference sequence name (NCBI accession or generic "reference")
2. `start` - Primer start coordinate (0-based, inclusive)
3. `stop` - Primer end coordinate (0-based, exclusive)  
4. `name` - Primer name following articbedversion naming conventions
5. `pool` - Pool assignment (1, 2, etc.) or BED score
6. `strand` - Strand orientation (+ for forward, - for reverse)
7. `sequence` - Primer sequence (5' to 3') with IUPAC support

#### **Version Differences**
- **v2.0 (Legacy)**: Simple naming (`SARS-CoV-2_1_LEFT`), generic reference names
- **v3.0 (Current)**: Enhanced naming support, NCBI accessions, header lines with `#`

#### **Naming Convention**
- Forward primers: `AMPLICON_LEFT`
- Reverse primers: `AMPLICON_RIGHT`
- Pool information encoded in amplicon names or separate file

#### **Example Usage**
```bash
# Convert ARTIC BED file
preprimer convert --input primers.scheme.bed --output-formats fasta sts

# Get information about ARTIC file
preprimer info existing_scheme.bed
```

### **Olivar Format**

**Description:** Native CSV format from [Olivar primer design tool](https://github.com/treangenlab/Olivar) using the SADDLE algorithm for variant-aware primer design with comprehensive amplicon metadata.

**File Extensions:** `.csv`, `olivar-design.csv`

**Format Characteristics:**
- Row-based amplicon format with forward and reverse primer pairs per row
- Complete metadata including amplicon sequences and coordinates
- 1-based coordinates with full amplicon span information  
- Full IUPAC degenerate nucleotide support for variant-aware designs
- Outputs both native CSV format and ARTIC-compatible BED format (`olivar-design.primer.bed`)
- Compatible with Olivar-generated configurations and JSON metadata
- Validated with real-world Olivar mitochondrial tiling schemes
- Topology-aware: supports circular genome designs with coordinate wrapping

#### **File Structure** (Complete Olivar format with metadata)
```csv
reference,amplicon_id,pool,fP,rP,start,insert_start,insert_end,end,amplicon,insert,fP_full,rP_full
NC_012920.1,NC_012920.1_1,1,cgctggagccggagca,ctaagcactctactcttagtttactgc,96,112,1427,1454,cgctggagccggagcaccctat...,ccctatgtcgcagtatctgtc...,cgctggagccggagca,ctaagcactctactcttagtttactgc
NC_012920.1,NC_012920.1_2,2,cggtcacacgattaacccaag,tttgggattttttaggtagtgggtg,900,921,2214,2239,cggtcacacgattaacccaagt...,tcaatagaagccggcgtaaag...,cggtcacacgattaacccaag,tttgggattttttaggtagtgggtg
HAV_1,HAV_1,1,TGGATRTTTCAGGWGTVCAAGC,TGWGGRAAATTMACTTTRGACCA,2225,2247,3394,3417,TGGATRTTTCAGGWGTVCAAGC...,TGWGGRAAATTMACTTTRGACCA...,TGGATRTTTCAGGWGTVCAAGC,TGWGGRAAATTMACTTTRGACCA
```

#### **Required Columns** (Official Olivar specification)
- `amplicon_id` - Unique amplicon identifier  
- `fP` - Forward primer sequence (5' to 3') with IUPAC support
- `rP` - Reverse primer sequence (5' to 3') with IUPAC support
- `pool` - Pool assignment (integer)

#### **Extended Columns** (Full Olivar output)
- `reference` - Reference sequence identifier (NCBI accession or name)
- `start` - Amplicon start coordinate (1-based)
- `end` - Amplicon end coordinate (1-based, inclusive)  
- `insert_start` - Insert region start coordinate
- `insert_end` - Insert region end coordinate
- `amplicon` - Complete amplicon sequence
- `insert` - Insert region sequence
- `fP_full` - Full forward primer sequence
- `rP_full` - Full reverse primer sequence

#### **Circular Genome Support**
- Handles coordinate wrapping (e.g., start=16400, end=200 for circular genomes)
- Topology detection for mitochondrial and plasmid designs
- Cross-origin amplicon support with proper length calculation

#### **Example Usage**
```bash
# Convert Olivar CSV
preprimer convert --input olivar-design.csv --output-formats artic fasta

# Check Olivar file details
preprimer info olivar_primers.csv
```

## 📤 **Output Formats**

PrePrimer converts to five different output formats for maximum compatibility and bidirectional conversion.

### **ARTIC Primerscheme Format**

**Description:** Official primerscheme structure following [primal-page specifications](https://github.com/artic-network/primal-page) with complete metadata support.

**File Extension:** `primer.bed` (part of complete scheme structure)

**Use Cases:**
- Direct use with `artic minion`, `artic medaka`, and amplicon-nf workflows
- Integration with [PrimerSchemes Labs](https://labs.primalscheme.com/) repository
- Nanopore and Illumina tiled amplicon sequencing
- Compatibility with entire ARTIC ecosystem

#### **Output Structure** (Official primerscheme format)
```
output_dir/
└── artic/
    ├── primer.bed           # 7-column extended BED format
    ├── reference.fasta      # Reference genome sequence  
    └── info.json           # Primal-page metadata with MD5 validation
```

#### **File Format** (7-column extended BED with sequence)
```bed
MN908947.3	47	78	SARS-CoV-2_1_LEFT	1	+	CTCTTGTAGATCTGTTCTCTAAACGAACTTT
MN908947.3	419	447	SARS-CoV-2_1_RIGHT	1	-	AAAACGCCTTTTTCAACTTCTACTAAGC
NC_012920.1	16399	16419	NC_012920.1_1_LEFT	1	+	GATCACAGGTCTATCACCCT
NC_012920.1	377	397	NC_012920.1_1_RIGHT	1	-	GCTGGAGCCGGAGCACCCTAT
```

#### **Metadata Support** (info.json with primal-page schema)
- Scheme versioning and authorship information
- MD5 hash validation for file integrity
- articbedversion compliance (v2.0/v3.0)
- Biological metadata (species, amplicon size, algorithm version)

#### **Features**
- 0-based BED coordinates
- Pool information encoded in score column
- Compatible with ARTIC directory structure
- Includes reference FASTA when available

#### **Official Primerscheme Structure**
```
scheme_directory/
├── primer.bed          # Primer coordinates (0-based BED format)
├── reference.fasta     # Reference genome sequence  
└── info.json          # Scheme metadata (following primal-page schema)
```

#### **Usage with Downstream Tools**
```bash
# Convert to primalscheme3-compatible format
preprimer convert --input primers.tsv --output-dir schemes/ --output-formats artic --prefix sars-cov-2

# Use with amplicon-nf pipeline
nextflow run nf-core/amplicon-nf --input samplesheet.csv --scheme schemes/sars-cov-2/

# Compatible with ARTIC workflows
artic minion sars-cov-2 data/ --scheme-directory schemes/
```

### **FASTA Format**

**Description:** Multi-FASTA format with detailed primer information in headers.

**File Extension:** `.fasta`

**Use Cases:**
- In-silico primer validation
- Sequence analysis and BLAST searches
- Custom bioinformatics pipelines
- Primer database creation

#### **Output Structure**
```
output_dir/
└── fasta/
    └── PROJECT_NAME.fasta
```

#### **File Format**
```fasta
>PROJECT_1_LEFT pool=1 amplicon=1 direction=forward start=100 stop=120
ATCGATCGATCGATCGATCG
>PROJECT_1_RIGHT pool=1 amplicon=1 direction=reverse start=380 stop=400
CGATCGATCGATCGATCGAT
>PROJECT_2_LEFT pool=2 amplicon=2 direction=forward start=250 stop=270
GCTAGCTAGCTAGCTAGCTA
>PROJECT_2_RIGHT pool=2 amplicon=2 direction=reverse start=530 stop=550
TAGCTAGCTAGCTAGCTAGC
```

#### **Header Information**
- `pool` - Pool assignment number
- `amplicon` - Amplicon identifier
- `direction` - Primer direction (forward/reverse)
- `start` - Start coordinate (1-based)
- `stop` - End coordinate (1-based)

#### **Customization Options**
```json
{
    "writers": {
        "fasta": {
            "line_width": 80,
            "include_pools": true,
            "include_coordinates": true,
            "header_format": ">{{name}} pool={{pool}} amplicon={{amplicon}}"
        }
    }
}
```

### **STS Format**

**Description:** Tab-separated format for Sequence Tagged Sites (STS) and in-silico PCR validation.

**File Extension:** `.sts.tsv`

**Use Cases:**
- In-silico PCR with e-PCR or me-pcr
- Amplicon prediction and validation
- Cross-reference with genome databases
- Quality control workflows

#### **Output Structure**
```
output_dir/
└── sts/
    └── PROJECT_NAME.sts.tsv
```

#### **File Format**
```tsv
primer_name	forward_primer	reverse_primer
PROJECT_1	ATCGATCGATCGATCGATCG	CGATCGATCGATCGATCGAT
PROJECT_2	GCTAGCTAGCTAGCTAGCTA	TAGCTAGCTAGCTAGCTAGC
PROJECT_3	TTTTAAAAAGGGGCCCCCC	CCCCGGGGAAAAATTTTTT
```

#### **Column Specification**
1. `primer_name` - Amplicon/primer pair name
2. `forward_primer` - Forward primer sequence (5' to 3')
3. `reverse_primer` - Reverse primer sequence (5' to 3')

#### **Usage with e-PCR**
```bash
# Convert to STS format
preprimer convert --input primers.tsv --output-formats sts --prefix MyVirus

# Use with e-PCR for validation
e-PCR -S reference.fasta -N primers.sts.tsv -o results.txt
```

### **VarVAMP TSV Format**

**Description:** VarVAMP-compatible tab-separated format for primer design workflows.

**File Extension:** `.tsv`

**Use Cases:**
- VarVAMP primer design tool input
- Primer scheme analysis and validation
- Cross-platform primer data exchange
- Research and development workflows

#### **Output Structure**
```
output_dir/
└── varvamp/
    └── PROJECT_NAME.tsv
```

#### **File Format**
```tsv
amplicon_name	primer_name	sequence	start	stop	length	gc_content	tm	penalty	pool
COVID_1	COVID_1_F	ATCGATCGATCGATCGATCG	100	120	20	0.5	60.2	1.5	1
COVID_1	COVID_1_R	CGATCGATCGATCGATCGAT	380	400	20	0.5	59.8	1.2	1
COVID_2	COVID_2_F	GCTAGCTAGCTAGCTAGCTA	250	270	20	0.5	60.5	1.8	2
COVID_2	COVID_2_R	TAGCTAGCTAGCTAGCTAGC	530	550	20	0.5	60.1	1.6	2
```

#### **Column Specification**
- `amplicon_name` - Amplicon identifier
- `primer_name` - Unique primer name
- `sequence` - Primer DNA sequence (5' to 3')
- `start` - Start coordinate (1-based)
- `stop` - End coordinate (1-based)
- `length` - Primer length in base pairs
- `gc_content` - GC content ratio (0.0-1.0)
- `tm` - Melting temperature in Celsius
- `penalty` - Primer design penalty score
- `pool` - Pool assignment number

## 🔍 **Format Detection**

PrePrimer automatically detects input formats based on:

### **File Extensions**
- `.tsv`, `.txt` → VarVAMP
- `.bed`, `.scheme.bed` → ARTIC
- `.csv` → Olivar

### **Header Analysis**
```python
# VarVAMP detection
headers = ["amplicon_name", "primer_name", "sequence", "start", "stop"]

# ARTIC detection  
bed_format = tab_separated and 6_columns and numeric_coordinates

# Olivar detection
headers = ["amplicon_id", "fP", "rP"] or similar_variations
```

### **Content Validation**
- Column count and types
- Coordinate format (0-based vs 1-based)
- Sequence validity
- Required field presence

### **Manual Format Override**
```bash
# Force specific format when detection fails
preprimer convert --input ambiguous.txt --format varvamp --output-formats artic

# Check what PrePrimer detected
preprimer info mystery_file.csv
```

## 📊 **Format Comparison**

| Feature | VarVAMP | ARTIC | Olivar |
|---------|---------|--------|--------|
| **Format** | TSV | BED | CSV |
| **Coordinates** | 1-based | 0-based | Variable |
| **Primer Layout** | One per row | One per row | Pair per row |
| **Pool Info** | Column | Name/separate | Column |
| **Quality Metrics** | Yes | No | Limited |
| **Reference** | Optional | Required | Optional |

### **Coordinate System Differences**

**VarVAMP (1-based):**
```
Sequence: ATCGATCG...
Position: 12345678...
Primer:   ^^^^      (start=1, stop=4)
```

**ARTIC BED (0-based):**
```  
Sequence: ATCGATCG...
Position: 01234567...
Primer:   ^^^^      (start=0, stop=4)
```

**Coordinate Conversion:**
```python
# VarVAMP to BED
bed_start = varvamp_start - 1
bed_stop = varvamp_stop  

# BED to VarVAMP
varvamp_start = bed_start + 1
varvamp_stop = bed_stop
```

## ⚠️ **Format Limitations**

### **VarVAMP Format**
- Requires specific column headers (case-sensitive by default)
- Tab-separated only (not space-separated)
- Pool information must be numeric
- Coordinates must be 1-based integers

### **ARTIC Format**
- Must follow BED format specification exactly
- Requires 6 columns minimum
- Strand column must be '+' or '-'
- Score column typically ignored but should be numeric

### **Olivar Format**
- Forward and reverse primers must be in same row
- Column names may vary (fP, rP, forward_primer, reverse_primer)
- Pool assignment may be missing (auto-assigned)
- Coordinates optional but helpful for validation

## 🔧 **Troubleshooting Format Issues**

### **Format Detection Failed**
```bash
# Problem: "Could not detect input format"
# Solution: Force format explicitly
preprimer convert --input file.txt --format varvamp --output-formats artic
```

### **Column Header Mismatches**  
```bash
# Problem: "Missing required column 'amplicon_name'"
# Solution: Check spelling, configure alternate headers
{
    "parsers": {
        "varvamp": {
            "header_variations": ["amplicon_name", "amlicon_name", "amplicon_id"]
        }
    }
}
```

### **Coordinate System Errors**
```bash
# Problem: "Invalid coordinates: start > stop"
# Solution: Check coordinate system (0-based vs 1-based)
preprimer info problematic_file.bed --detailed
```

### **Genome Topology Detection**

PrePrimer automatically detects whether genomes are linear or circular for biologically accurate interpretation.

#### **Topology Detection Methods**

**1. Metadata File Detection:**
```yaml
# metadata.yaml in same directory as primer file
topology: circular
genome_length: 16569
reference_id: NC_012920.1
organism: Homo sapiens mitochondrion
```

**2. Known Reference Detection:**
```bash
# Automatically detected topologies for common references
# NC_012920.1 → circular (human mitochondrial genome, 16,569 bp)
# NC_045512.2 → linear (SARS-CoV-2 genome, 29,903 bp)
# LR722600.1 → linear (ASFV genome, 191,232 bp)
```

**3. Organism Pattern Detection:**
```bash
# Patterns that suggest circular topology:
# "mitochondrion", "mitochondrial", "chloroplast", "plastid"
```

#### **Circular Genome Coordinate Handling**

**Valid Wrapping Coordinates:**
```tsv
# Mitochondrial genome (16,569 bp) - coordinates can wrap around
amplicon_name	primer_name	sequence	start	stop	pool
NC_012920.1_1	NC_012920.1_1_FW	ATCGATCG...	16400	16420	1
NC_012920.1_1	NC_012920.1_1_RW	CGATCGAT...	200	220	1
```

**Topology-Aware Length Calculation:**
```python
# Linear genome: 16420 - 200 = 16220 bp (biologically impossible for PCR)
# Circular genome: (16569 - 16400) + 200 + 1 = 370 bp (valid amplicon)
```

**Automatic Validation Warnings:**
```bash
# PrePrimer warns about potential topology mismatches
preprimer convert --input primers.tsv --output-formats artic
# WARNING: High start (16400) with low end (200) on linear genome - check topology
```

**Human Mitochondrial DNA (Circular):**
